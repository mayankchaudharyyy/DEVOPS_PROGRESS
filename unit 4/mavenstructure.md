# Maven Directory Structure

Maven follows a **Convention Over Configuration** approach, which means it expects projects to follow a standard directory layout. By following these conventions, developers can reduce configuration and focus more on development.

## Standard Maven Structure

```text
my-app/
│
├── pom.xml
├── src/
│   ├── main/
│   │   ├── java/
│   │   └── resources/
│   │
│   └── test/
│       ├── java/
│       └── resources/
│
├── target/
├── README.md
└── .gitignore
```

## Important Directories

### src/main/java

Contains the main application source code such as classes, services, controllers, models, and utility files.

Examples:

* Main.java
* UserService.java
* OrderController.java

Compiled files are stored in `target/classes`.

### src/main/resources

Contains configuration and resource files required by the application.

Examples:

* application.properties
* application.yml
* logback.xml
* SQL scripts
* Templates and static files

These resources are included in the final JAR/WAR file.

### src/test/java

Contains unit tests and integration tests used to verify application functionality.

Examples:

* UserServiceTest.java
* OrderControllerTest.java

Compiled test files are stored in `target/test-classes` and are not included in the final build.

### src/test/resources

Stores test-specific configuration files, sample data, and test resources.

Examples:

* application-test.properties
* test-data.sql

### target

This directory is automatically generated by Maven during the build process.

It contains:

* Compiled `.class` files
* Generated JAR/WAR files
* Test reports
* Other build artifacts

## Key Benefits

* Standardized project structure
* Easy project maintenance
* Better team collaboration
* Simplified build and dependency management
* Reduced configuration effort

## Key Takeaway

Maven's directory structure provides a well-organized layout for source code, resources, tests, and build outputs. Following this structure makes projects easier to build, maintain, and manage.
