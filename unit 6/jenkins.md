# Unit VI — Jenkins: Basics & Concepts

## What is Jenkins?

Jenkins is an open-source automation tool used to automate software development tasks such as building, testing, and deploying applications.

It is mainly used for Continuous Integration (CI) and Continuous Delivery (CD).

---

## Why Use Jenkins?

* Automates builds and testing
* Reduces manual work
* Integrates with tools like Git, Docker, Maven, and Kubernetes
* Supports complete CI/CD pipelines
* Large plugin ecosystem
* Works on Linux, Windows, and macOS

---

## Jenkins Key Terms

### Job / Project

A task executed by Jenkins, such as building or testing an application.

### Build

A single execution of a Jenkins job.

### Node

A machine where Jenkins runs jobs.

* Master: Controls Jenkins
* Agent: Executes tasks

### Pipeline

A sequence of automated steps that define the CI/CD workflow.

### Plugin

An extension that adds extra functionality to Jenkins.
Examples: Git Plugin, Docker Plugin, Maven Plugin.

---

## Advantages of Jenkins

* Open source and free
* Easy integration with DevOps tools
* Supports automation at every stage
* Saves time and improves productivity
* Easy to scale using multiple agents

---

## Summary

Jenkins is one of the most popular CI/CD tools used to automate software development workflows. It helps teams build, test, and deploy applications faster and more reliably.
